# Документация по интерфейсу (UX/UI Guidelines)

Этот раздел документации включает базовые принципы проектирования интерфейсов в проекте, а также разбор частых паттернов поведения пользователей. Данное руководство необходимо прочитать дизайнерам и фронтенд-разработчикам.

## 1. Цветовая палитра и Тематика
Интерфейс построен на утилитах Tailwind CSS с использованием стандартной и расширенной палитры.
- **Primary Tone**: Цвет `Indigo` (от `indigo-50` до `indigo-700`). Используется для основных акцентных кнопок и ссылок.
- **Secondary Tone**: Цвета `Slate` и `Gray`. Фоновый слой — `slate-50`, карточки — белые, обводки — `slate-200`. Это обеспечивает "чистоту" и легкость интерфейса.
- **Семантические цвета (Оповещения/Статусы)**:
  - Успех (Выполненные заказы, оплата, тосты-сообщения): `Emerald` (`emerald-500` / `emerald-600`).
  - Ошибка/Удаление/Отмена (Отказанные заказы): `Red` (`red-500` / `red-600`).
  - Предупреждение (Неоплаченные, ожидающие): `Amber` / `Yellow`.

## 2. Типографика
Шрифты и размеры определены также классами Tailwind. Приоритет отдается удобочитаемости для дисплеев телефонов.
- Названия экранов / модалок: `text-xl font-bold` до `text-2xl font-black`.
- Текст строк таблиц с клиентами и авто: `text-sm font-semibold`.
- Мелкие справочные данные (время, айди): `text-xs font-normal text-slate-500`.

## 3. Компоненты взаимодействия (Паттерны)

### 3.1. Мобильная Навигация (Bottom Bar)
Поскольку основные юзеры (Мастера) работают "на ногах" и с телефоном:
- Боковое меню `Sidebar.vue` на мобилках скрывается.
- Вместо него отображается `MobileNav.vue` — нижняя фиксированная панель, до всех пунктов которой можно дотянуться большим пальцем правой руки.

### 3.2. Модальные окна (Свайпы и Закрытия)
Формы (например, создание записи `RecordModal`) открываются как карточки оверлея поверх экрана.
- **Закрытие**: Клик по зоне вне окна (`@click="close"` на обертке backdrop) не должен закрывать форму, если в ней есть несохраненные изменения (это может привести к потере данных, когда клиент случайно кликнул мимо формы).
- **Скроллинг внутри длинных списков (Марки, Модели)**:
  *Критическое правило разработки в нашем приложении:* Для списков выбора следует использовать событие `@click.stop`, а *не* `@mousedown` / `@touchstart`. \
  **Почему:** Если юзер пытается прокрутить список марок машин на мобильном устройстве через свайп, события `mousedown`/`touchstart` вызываются в момент касания пальцем — и сразу же выбирают элемент, под которым был палец. `@click` же учитывает отпускание пальца, и встроенные браузерные движки блокируют клик, если это был жест `scroll` свайпом. Этот баг учтен и исправлен во всех дропдаунах.

### 3.3. Кнопки сохранения и загрузки
При отправке данных кнопки должны блокироваться (переходить в disable state) и показывать спиннер или текст "Загрузка/Синхронизация", чтобы избежать дабл-клик спама (отправки нескольких заявок).

## 4. Геймификация и Игровой интерфейс
Раздел "Игровизации" интегрирован в систему бесшовно.
- Игры рисуются либо поверх Canvas 2D Engine (Rider, Snake), либо в компонентах Vue (2048).
- UI Игр отделен от CRM: используются неоновые, темные стили (Cyberpunk, Arcade).
- Для Игр приоритетно скрывать полосу прокрутки (`overflow: hidden` у body/главного контейнера) и обеспечивать `touch-action: none` / перехват кнопок-стрелок, чтобы при игре экран телефона/ПК не прокручивался вверх-вниз случайным образом.
