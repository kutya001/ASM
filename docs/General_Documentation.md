# Общая документация

## Описание системы
AutoCRM — это гибридное веб-приложение (Single Page Application), предназначенное для управления станциями технического обслуживания автомобилей (СТО). Приложение решает задачи электронного документооборота, диспетчеризации заявок и хранения справочников.

Особенностью приложения является наличие **Игровизации (Gamification)** — встроенный раздел `Игротека` содержит коллекцию мини-игр для отдыха сотрудников.

## Идея и архитектурные решения

Проект построен как **frontend-driven** приложение с использованием стейт-менеджера для хранения состояния и системы очередей (syncQueue) для обеспечения оптимистичного UI.

### Оптимистичный UI (Optimistic Updates)
Любое действие (сохранение записи, перевод статуса, добавление справочника) сначала применяется локально в стейте клиента, после чего отправляется запрос на сервер. В случае ошибки локальный стейт откатывается/выводится сообщение. Это создает ощущение мгновенной реакции приложения на действия пользователя (нулевой лаг).

## Основные разделы
1. **Авторизация (AuthView)** — Вход по логину (номеру) и паролю в зависимости от роли.
2. **Дашборд (DashboardTab)** — Краткая сводка, список текущих записей, доходы, статистика по клиентам.
3. **Записи (RecordsTab)** — Основной рабочий экран в виде таблицы с возможностью фильтрации по датам, мастерам, брендам, статусам, моделям авто.
4. **Заявки (ApplicationsTab)** — Управление входящими или планируемыми заявками на обслуживание.
5. **Справочники (RefsTab)** — Раздел для управления каталогом услуг, марками и моделями автомобилей (Только для администраторов).
6. **Пользователи (UsersTab)** — Управление списком мастеров и сотрудников (Только для Superadmin).
7. **Игротека (GameContainer)** — Каталог развлекательных модулей.

## Инструменты развертывания
Данный проект настроен на бесперебойную сборку посредством автоматизированных систем CI/CD. 
Для компиляции приложения используется `npm run build` (на базе Vite). 
Результирующий каталог `dist` может быть отправлен на любой статический хостинг или запакован в Docker-контейнер и развернут, например, в Google Cloud Run.
